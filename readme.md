# 基于大模型的智能医疗随访系统

## 项目背景

针对传统医疗随访人工成本高、患者档案非结构化、问答检索困难等痛点，设计并实现了一套基于大模型的自动化随访系统，实现"语音问诊-智能解答-档案生成"的全流程闭环，为医疗随访工作提供智能化解决方案。

## 项目功能

- **智能问卷生成**：根据医疗需求自动生成结构化问卷
- **自动化随访**：基于大模型的智能问答，引导患者完成随访流程
- **健康评估**：根据患者反馈生成健康风险评估和建议
- **档案管理**：自动生成结构化随访记录和JSON格式档案
- **知识增强**：采用RAG技术，从健康知识库中检索信息，提供更准确的健康建议

## 技术架构

- **前端**：命令行交互界面
- **后端**：Python + LangChain + 大模型API
- **存储**：文件系统存储（支持扩展为数据库存储）
- **知识增强**：FAISS向量存储 + OpenAI Embeddings

## 快速开始

### 环境准备

1. **创建虚拟环境**
   ```bash
   python3 -m venv venv
   ```

2. **激活虚拟环境**
   ```bash
   source venv/bin/activate  # Linux/Mac
   # 或
   venv\Scripts\activate  # Windows
   ```

3. **安装依赖**
   ```bash
   pip install -r requirements.txt
   ```

4. **配置环境变量**
   创建 `.env` 文件，添加以下内容：
   ```
   KIMI_API_KEY=your_kimi_api_key
   KIMI_API_BASE=your_kimi_api_base
   DASHSCOPE_API_KEY=your_dashscope_api_key
   QWEN_API_BASE=your_qwen_api_base
   ```

### 使用流程

#### 1. 生成问卷

当有新的问卷需求时，将PDF格式的问卷放入 `design/` 目录，然后运行：

```bash
python design/designer.py
```

生成的问卷将保存到 `Q.txt` 文件中。

#### 2. 执行随访

运行问卷调查系统：

```bash
python -m survey.main
```

系统将：
- 引导患者完成问卷回答
- 实时生成随访记录
- 提供健康评估和建议
- 保存随访记录到 `final.txt` 和 `final1.json` 文件

## 核心技术

1. **大模型集成**：通过LangChain封装多种大模型API，实现统一的交互接口
2. **RAG技术**：结合向量存储和知识检索，增强模型的知识能力
3. **对话管理**：实现上下文感知的智能对话，提供流畅的随访体验
4. **多格式输出**：支持文本和JSON格式的随访记录，方便后续处理和分析
