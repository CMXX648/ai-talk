# 随访系统与Django后端平台联动使用说明

## 系统概述

本系统实现了随访调查系统与Django后端平台的联动，具有以下功能：
1. 随访开始前从Django数据库获取患者信息
2. 根据患者性别生成个性化称呼（先生/女士）
3. 随访结束后将随访记录保存到Django数据库

## 使用步骤

### 1. 启动Django后端平台

```bash
cd /root/talk/medical_followup
python manage.py runserver 8001
```

访问地址：
- 管理后台：http://127.0.0.1:8001/admin/
- 前端页面：http://127.0.0.1:8001/

### 2. 添加患者和医生信息

#### 添加患者信息
1. 访问 http://127.0.0.1:8001/admin/
2. 登录管理后台
3. 进入 "Patients" 模块
4. 点击 "Add patient"
5. 填写患者信息：
   - 姓名
   - 性别（男/女）
   - 年龄
   - 电话号码

#### 添加医生信息
1. 访问 http://127.0.0.1:8001/admin/
2. 进入 "Doctors" 模块
3. 点击 "Add doctor"
4. 填写医生信息：
   - 姓名
   - 性别
   - 电话号码
   - 科室（可选）
   - 职称（可选）

### 3. 配置医生信息

编辑 `/root/talk/config.py` 文件，配置医生信息：

```python
DOCTORS = [
    {
        'name': '王医生',
        'gender': '男',
        'phone': '13800000000',
        'department': '内科',
        'title': '主治医师'
    },
    {
        'name': '刘医生',
        'gender': '女',
        'phone': '13900000000',
        'department': '心内科',
        'title': '副主任医师'
    }
]
```

**注意**：确保配置文件中的医生电话号码与Django数据库中的医生电话号码一致。

### 4. 运行随访系统

```bash
cd /root/talk
python -m survey.main
```

### 5. 随访流程

#### 步骤1：选择患者
系统会显示数据库中的患者列表：
```
=== 医疗随访系统 ===
请选择患者进行随访：
1. 张三 (男, 45岁)
2. 李四 (女, 38岁)
3. 王五 (男, 67岁)
请输入患者编号（输入0直接开始随访）: 
```

#### 步骤2：个性化问候
如果选择了患者，系统会根据患者性别生成个性化问候：
```
AI: 您好，张三先生！
```

#### 步骤3：进行随访
系统会按照问卷问题顺序进行随访，收集患者健康信息。

#### 步骤4：选择医生
随访结束后，系统会显示医生列表供选择：
```
=== 保存随访记录 ===
请选择负责本次随访的医生：
1. 王医生 (内科, 主治医师)
2. 刘医生 (心内科, 副主任医师)
请输入医生编号（默认为1）: 
```

#### 步骤5：保存随访记录
系统会将随访记录保存到Django数据库：
```
随访记录已保存到数据库，记录ID: 1
患者: 张三
医生: 王医生
随访时间: 2026-03-15 14:30:00
```

### 6. 查看随访记录

#### 通过管理后台查看
1. 访问 http://127.0.0.1:8001/admin/
2. 进入 "Follow up records" 模块
3. 查看所有随访记录

#### 通过前端页面查看
1. 访问 http://127.0.0.1:8001/patients/
2. 点击患者姓名
3. 查看该患者的所有随访记录

## API接口说明

### django_api.py 提供的接口

#### 获取患者信息
```python
from django_api import get_patient_by_name, get_all_patients

# 根据姓名获取患者信息
patient_info = get_patient_by_name("张三")

# 获取所有患者信息
patients = get_all_patients()
```

#### 保存随访记录
```python
from django_api import save_follow_up_record

result = save_follow_up_record(
    patient_phone="13800138001",
    doctor_phone="13600136001",
    content="随访内容",
    health_assessment="健康评估",
    recommendations="建议措施"
)
```

#### 获取患者随访记录
```python
from django_api import get_patient_records

records = get_patient_records("13800138001")
```

## 注意事项

1. **数据库连接**：确保Django数据库连接正常，随访系统才能正常工作。

2. **电话号码一致性**：确保配置文件中的医生电话号码与Django数据库中的医生电话号码一致。

3. **环境变量**：确保.env文件中的API密钥配置正确。

4. **并发访问**：如果多个用户同时使用随访系统，可能会出现数据库并发问题，建议添加适当的锁机制。

5. **错误处理**：如果保存随访记录失败，系统会显示错误信息，但不会影响随访流程。

## 故障排除

### 问题1：无法连接到Django数据库
**解决方案**：
- 检查Django服务是否正在运行
- 检查数据库配置是否正确
- 检查数据库文件权限

### 问题2：患者列表为空
**解决方案**：
- 确保已在Django管理后台添加患者信息
- 检查数据库连接是否正常

### 问题3：保存随访记录失败
**解决方案**：
- 检查医生电话号码是否正确
- 检查患者电话号码是否正确
- 检查数据库连接是否正常

### 问题4：个性化问候不正确
**解决方案**：
- 检查患者性别信息是否正确
- 检查性别字段是否为"男"或"女"
